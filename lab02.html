如果你在 Python 中输入以下代码，会发生什么？

1 / 0

在 Python 中试试！你会看到一个 ZeroDivisionError。但是下面这个表达式呢？

True or 1 / 0

它求值为 True，因为 Python 的 and 和 or 运算符会短路求值（short-circuit）。也就是说，它们不一定会对每个操作数求值。

运算符 检查是否： 从左到右求值到： 示例
AND 所有值都为真 第一个假值 False and 1 / 0 求值为 False
OR 至少一个值为真 第一个真值 True or 1 / 0 求值为 True

短路求值发生在运算符遇到一个能让它得出结论的操作数时。例如，and 一旦遇到第一个假值就会短路，因为它知道并非所有值都为真。

如果 and 和 or 不短路，它们就返回最后一个值；另一种记忆方式是：and 和 or 总是返回它们最后求值的对象，无论是否短路。请记住，当使用 True 和 False 以外的值时，and 和 or 不一定返回布尔值。

变量是绑定到值的名称，值可以是原始类型如 3 或 'Hello World'，也可以是函数。由于函数可以接受任何值作为参数，其他函数也可以作为参数传入。这就是高阶函数的基础。

高阶函数（higher-order function）是一类操作其他函数的函数：它可以将函数作为参数接收，也可以返回一个函数，或两者兼有。本实验将介绍高阶函数的基础知识，下一次实验将深入探索其更多应用。

函数作为参数

在 Python 中，函数对象是可以被传递的值。我们知道创建函数的一种方式是使用 def 语句：

def square(x): return x * x

上面的语句创建了一个内在名称为 square 的函数对象，并将其绑定到当前环境中的名称 square。现在让我们尝试将其作为参数传递。

首先，编写一个接收另一个函数作为参数的函数：

def scale(f, x, k): """ Returns the result of f(x) scaled by k. """ return k * f(x)

现在我们可以用 square 和其他参数调用 scale：

>>> scale(square, 3, 2) # Double square(3) 18 >>> scale(square, 2, 5) # 5 times 2 squared 20

注意，在调用 scale 的函数体中，内在名称为 square 的函数对象被绑定到参数 f。然后我们在 scale 的函数体中调用 f(x) 来调用 square。

正如我们在上面的 lambda 部分中看到的，我们也可以将 lambda 表达式传入调用表达式！

>>> scale(lambda x: x + 10, 5, 2) 30

在这个调用表达式的帧中，名称 f 被绑定到由 lambda 表达式 lambda x: x + 10 创建的函数。

返回函数的函数

因为函数也是值，它们可以作为返回值！以下是一个例子：

def multiply_by(m): def multiply(n): return n * m return multiply

在这个例子中，我们在 multiply_by 的函数体内定义了 multiply 函数，然后将其返回。让我们看看它的用法：

>>> multiply_by(3) <function multiply_by.<locals>.multiply at ...> >>> multiply(4) Traceback (most recent call last): File "<stdin>", line 1, in <module> NameError: name 'multiply' is not defined

调用 multiply_by 返回一个函数，正如预期。然而，调用 multiply 会出错，即使这是内部函数的名字。这是因为名称 multiply 只存在于求值 multiply_by 函数体的帧中。

那我们如何使用这个内部函数呢？以下是两种方式：

>>> times_three = multiply_by(3) # 将调用表达式的结果赋值给一个名称 >>> times_three(5) # 用新名称调用内部函数 15 >>> multiply_by(3)(10) # 将两个调用表达式链接在一起 30

关键是，因为 multiply_by 返回一个函数，你可以像使用任何其他函数一样使用它的返回值。

Lambda 表达式是通过指定两件事来求值为函数的表达式：参数和返回表达式。

lambda <parameters>: <return expression>

虽然 lambda 表达式和 def 语句都创建函数对象，但它们有一些显著的区别。lambda 表达式的工作方式与其他表达式类似；就像数学表达式仅仅求值为一个数字而不改变当前环境一样，lambda 表达式求值为一个函数而不改变当前环境。让我们仔细看一下。

lambda def
类型 求值为一个值的表达式。 改变环境的语句。
执行结果 创建一个没有内在名称的匿名 lambda 函数。 创建一个有内在名称的函数，并将其绑定到当前环境中的该名称。
对环境的影响 求值 lambda 表达式不会创建或修改任何变量。 执行 def 语句会创建一个新的函数对象并将其绑定到当前环境中的一个名称。
用法 lambda 表达式可以在任何需要表达式的地方使用，例如在赋值语句中，或作为调用表达式的运算符或操作数。 执行 def 语句后，创建的函数被绑定到一个名称。你应该在需要表达式的任何地方使用这个名称来引用该函数。
示例
# lambda 表达式本身不会改变环境 lambda x: x * x # 我们可以用赋值语句将 lambda 函数赋值给名称 square = lambda x: x * x square(3) # lambda 表达式可作为运算符或操作数 negate = lambda f, x: -f(x) negate(lambda x: x * x, 3)
def square(x): return x * x # 由 def 语句创建的函数 # 可以通过其内在名称引用 square(3)

Q5: Count Cond

考虑以下使用 sum_digits 和 is_prime 函数的 count_fives 和 count_primes 实现：

def count_fives(n): """Return the number of values i from 1 to n (including n) where sum_digits(n * i) is 5. >>> count_fives(10) # Among 10, 20, 30, ..., 100, only 50 (10 * 5) has digit sum 5 1 >>> count_fives(50) # 50 (50 * 1), 500 (50 * 10), 1400 (50 * 28), 2300 (50 * 46) 4 """ i = 1 count = 0 while i <= n: if sum_digits(n * i) == 5: count += 1 i += 1 return count def count_primes(n): """Return the number of prime numbers up to and including n. >>> count_primes(6) # 2, 3, 5 3 >>> count_primes(13) # 2, 3, 5, 7, 11, 13 6 """ i = 1 count = 0 while i <= n: if is_prime(i): count += 1 i += 1 return count

这两个实现看起来非常相似！通过编写一个函数 count_cond 来泛化这个逻辑，它接收一个双参数谓词函数 condition(n, i)。count_cond 返回一个单参数函数，接收 n，统计从 1 到 n 中所有满足 condition 的数字个数。

💡 注意
当我们说 condition 是一个谓词函数（predicate function）时，我们指的是它返回 True 或 False 的函数。
def sum_digits(y): """Return the sum of the digits of non-negative integer y.""" total = 0 while y > 0: total, y = total + y % 10, y // 10 return total def is_prime(n): """Return whether positive integer n is prime.""" if n == 1: return False k = 2 while k < n: if n % k == 0: return False k += 1 return True def count_cond(condition): """Returns a function with one parameter N that counts all the numbers from 1 to n that satisfy the two-argument predicate function Condition, where the first argument for condition is n and the second argument is the number from 1 to n. >>> count_fives = count_cond(lambda n, i: sum_digits(n * i) == 5) >>> count_fives(10) # 50 (10 * 5) 1 >>> count_fives(50) # 50 (50 * 1), 500 (50 * 10), 1400 (50 * 28), 2300 (50 * 46) 4 >>> is_i_prime = lambda n, i: is_prime(i) # need to pass 2-argument function into count_cond >>> count_primes = count_cond(is_i_prime) >>> count_primes(2) # 2 1 >>> count_primes(3) # 2, 3 2 >>> count_primes(4) # 2, 3 2 >>> count_primes(5) # 2, 3, 5 3 >>> count_primes(20) # 2, 3, 5, 7, 11, 13, 17, 19 8 """ "*** YOUR CODE HERE ***"
